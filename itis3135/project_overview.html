Project Overview

Project Overview

Purpose: My clients application will be based off of design number 4. Design number 4 is to develop a webpage for a student who would like to learn the basics of JavaScript, CSS, HTML, etc. The goal is to provide a useful webpage specifically designed towards my clients needs and to allow them to dive into web development. This will allow my client to get a intro start on web-development to determine their interest in this path.

Intended Users: The intended users for this website would be any UNC Charlotte student that is currently not enrolled in any web-development focused course. This is to allow these students to get a quick grasp and try to determine if they would like to further continue this path of interest.

Content Overview: The overview of the content of the website that I will be developing for my client will consist of a landing page, about page, and language pages. Each language will consist of their own links inside of the navigation bar. Each language page will have the same layouts by containing sections. For example the first section on each page will consist of having a introduction, syntax, main content, examples, etc. Within each one of those pages a navbar will be included for moving down to certain sections of the page because it could be long depending on the content.

HTML Page Design

Page Name: HTML Page

Page Purpose: The purpose of this page is to help guide the user into learning the very basics of HTML.

Audience: UNC Charlotte Students

Page Content: The page will contain content of HTML basics, syntax, best practices, examples, images, etc.

User Entering Data? No

Validations? No

Contains Buttons/Hyperlinks/Dropdowns? Yes, contains hyperlinks and will also contain buttons to show and hide the table of contents for more reading space. There will also be the ability for each section of the page to minimize/hide the content and just keep the section name. If the user wants to re-open it they just need to click on the section title of whichever section they want shown. Lastly there will be a image gallery within each section showing the examples and will only display if the user clicks because otherwise it will take up a lot of the page.

What Actions Will Occur? When the user access the navbar hyperlinks/buttons it will navigate the user to other pages within the web application. If the user clicks on the arrow icon to hide the table of contents per language page, then the table of contents will be hidden and any excess room for the content will be re-shifted to left and still centered and so on. Once the user either clicks a hyperlink in the table of contents or if they click on a specific section title then all other sections content will be hidden but the titles and the only section content visible will be the one the user clicked.

Dynamic Functionality

The dynamic functionality within the website will consist of show/hide for content sections within the main as well as pages that have a table of contents. There will also be show/hide for image galleries within the sections on each language basics page. Lastly there will be a user-input text-entry within the home page.

The purpose of using the show/hide features are to help the user when reading to not get lost with all the clustering of content and to keep to page cleaner and more visible for reading. These features will only be usable within the pages of languages like the HTML, CSS, JavaScript, and JQuery pages.

The user-input text-entry on the home page will be specifically used for instead of using the navbar features the user can type in the language and it will redirect them to that page only if it exists.
